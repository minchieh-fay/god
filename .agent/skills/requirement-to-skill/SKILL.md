# 需求转化为施工图 (Demand Distillation)

## 目标
将模糊的需求描述转化为下游 AI（Cursor）可 100% 成功执行的 `SKILL.md`。

## 核心逻辑
1. **[上下文检索]**: 向上查找 `ARCH.md`。如果没有找到，必须报错停止。
2. **[冲突检查]**: 检查新需求是否违反了 `ARCH.md` 里的规范。
3. **[任务原子化]**: 
    - 任何一个 Task 涉及的代码修改量不应超过 50 行。
    - 如果超过，必须拆分为多个 Task。
4. **[防御性提醒]**: 针对当前项目技术栈，列出 3 个最容易出错的坑（如：忘记关闭文件流、内存泄漏风险、类型推导错误）。

## 输出格式
- 必须包含 `[Context]`, `[Blueprint]`, `[Implementation Steps]` 模块。
