# ARCH.md 规范

`ARCH.md` 是项目的技术宪法，由项目分析技能自动生成，必须涵盖：

- 整体架构设计与分层模式
- 目录结构说明
- 开发语言与核心依赖库
- 错误处理模式
- **禁止项（自动推断）**：通过观察代码，识别出作者从未采用的写法，列为禁止项

---

## ARCH.md 末尾必须包含的用户补充模块

生成 ARCH.md 时，末尾必须保留以下模块（内容留空，由用户自行填写）：

```markdown
## ⚠️ 用户补充说明（最高优先级，必须严格遵守）

> 本节由用户手动维护，内容凌驾于所有自动分析结论之上。
> 生成任何 SKILL.md 前，必须完整阅读本节，违反任意一条均视为输出无效。

### 禁止事项
<!-- 例：禁止引入任何 Python 脚本 -->
<!-- 例：禁止使用 ORM，所有数据库操作必须使用原生 SQL -->

### 注意事项
<!-- 例：所有对外接口必须做鉴权 -->
<!-- 例：日志必须使用项目统一的 logger，禁止使用 fmt.Println -->

### 框架说明
<!-- 例：HTTP 框架统一使用 Gin，禁止引入其他 HTTP 库 -->

### 业务说明
<!-- 例：用户 ID 全局唯一，由网关统一生成，服务内不得自行生成 -->

### 其他补充
<!-- 任何不属于以上分类但需要 AI 知晓的信息 -->
```

**生成 `SKILL.md` 前，必须完整读取此模块。此模块任意一条的优先级高于 ARCH.md 其他所有内容。**
