# 记忆机制

## 记忆文件位置

记忆分两个层级：

- **工程级记忆**：`projects/{project_name}/MEMORY.md`
  记录与该项目技术相关的教训（如：误用了禁止的库、路径判断错误等）

- **工作流级记忆**：`MEMORY.md`（工程根目录）
  记录与流程、沟通、分析方式相关的教训（如：需求追问不够、方案确认被跳过等）

---

## 启动时：主动读取记忆

每次开始处理需求前，必须按顺序读取：
1. 工程根目录的 `MEMORY.md`（如存在）
2. 当前项目的 `projects/{project_name}/MEMORY.md`（如存在）

读取后在心里执行一次扫描：**"这次需求是否与历史教训有交集？"**，有则主动规避。

---

## 被纠正时：触发自我反思

当用户表达纠正（如："不对"、"你搞错了"、"这不是我要的"、"上次你也犯过这个错"），必须：

1. 先理解并承认错误
2. 判断这个错误是否属于以下任一情形：
   - 同类错误**可能再次发生**
   - 错误源于对项目技术栈的**误判**
   - 错误源于对工作流程的**理解偏差**
3. 如果符合，主动询问用户：
   > "我是否应该把这个教训记录到记忆文件，避免下次重犯？"
4. 用户确认后，调用 `lesson-learned` 技能执行记录。

---

## 记忆的优先级

MEMORY.md 中的教训，优先级等同于 ARCH.md 的「用户补充说明」，生成 SKILL.md 前必须核查。
