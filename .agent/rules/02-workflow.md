# 完整工作流程

## Step 1：接收需求，确认项目

用户说："我有个需求是……"

**判断项目是否存在于 `projects/` 下：**

- **不存在**：
  1. 询问用户是否创建该项目
  2. 若用户确认，在 `projects/{project_name}/` 下创建目录
  3. 询问是否需要分析项目源码
  4. 若需要，让用户提供本地源码路径，执行项目分析，生成 `ARCH.md`

- **已存在，但缺少 `ARCH.md`**：
  1. 提醒用户该项目尚未建立技术档案
  2. 询问是否需要分析项目源码
  3. 若需要，让用户提供本地源码路径，执行项目分析，生成 `ARCH.md`

- **已存在，且有 `ARCH.md`**：直接进入 Step 2

---

## Step 2：归档需求文件

将需求写入文件：
```
projects/{project_name}/{YYYYMM}/{DD}/{需求名}.md
```

对需求进行初步分析——如果存在明显的信息缺失，**必须先向用户追问**，补全后再继续。

---

## Step 3：生成施工图

读取该项目的 `projects/{project_name}/ARCH.md`，结合用户需求，生成：
```
projects/{project_name}/{YYYYMM}/{DD}/{需求名}/SKILL.md
```

生成前，必须先向用户口头确认：
> "我已读取 {project_name} 的 ARCH.md，针对「{需求名}」需求，我计划采用 {XX方案}，是否开始生成施工图？"
