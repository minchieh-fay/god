# ARCH.md — Sample 项目技术档案

## 开发语言 / 技术栈
Go 1.21

## 整体架构设计与分层模式
Clean Architecture 三层结构：
- `handler`：HTTP 入口，负责请求解析与响应格式化
- `service`：业务逻辑层，核心规则在此处理
- `repo`：数据访问层，封装所有数据库操作

## 目录结构说明
```
~/
├── cmd/main.go          # 程序入口
├── internal/
│   ├── handler/         # HTTP 处理器
│   ├── service/         # 业务逻辑
│   └── repo/            # 数据库操作
├── pkg/
│   └── mailer/          # 邮件发送工具包
├── config/              # 配置文件
└── go.mod
```

## 核心依赖库
- `gin`：HTTP 框架
- `gorm`：ORM（仅限 MySQL）
- `zap`：日志库
- `viper`：配置管理

## 错误处理模式
统一返回结构：`{"code": 0, "msg": "ok", "data": {}}`
错误码定义在 `pkg/errcode/` 下，禁止在 handler 层直接返回裸错误字符串。

## 禁止项（自动推断）
- 禁止在 service 层直接操作 HTTP 上下文
- 禁止使用 `fmt.Println` 打印日志，必须使用 `zap`
- 禁止在 repo 层写业务逻辑
